QUICK START (oMLX, local, optimized for Mac)

oMLX provides optimized LLM inference for Apple Silicon and Intel Macs.
No network egress, local-only inference.

1. Install oMLX: https://github.com/jundot/omlx
2. Start the oMLX server (follow oMLX documentation for your setup).
3. Generate an admin API key in the oMLX admin interface.
4. Store the admin key in `~/.config/sublime-omlx/config.json` under `providers.omlx.api_key`:
   ```json
   "omlx": {
     "api_key": "your-admin-key-from-omlx-admin-interface",
     "base_url": "http://localhost:8000/v1"
   }
   ```
   Do not set a `model` field here — use `oMLX: Choose Model` in Sublime instead.
5. In Sublime: Tools -> oMLX: Open Chat (or use command palette: `oMLX: Open Chat`).
6. Type a message after the `<user> ` prompt.
7. Press Ctrl+Enter (macOS: Cmd+Enter) to send.

SETTINGS

Overrides in Packages/User/oMLX.sublime-settings.

HOSTED PROVIDERS

Set "provider" to "openai", "anthropic", "openrouter", "deepseek", or "custom".
API keys: use env vars (OPENAI_API_KEY, ANTHROPIC_API_KEY, OPENROUTER_API_KEY,
DEEPSEEK_API_KEY, CUSTOM_API_KEY) or the external config file at
~/.config/sublime-omlx/config.json (macOS/Linux) or
%APPDATA%\sublime-omlx\config.json (Windows).

DO NOT commit your oMLX.sublime-settings if you set keys there. The
plugin refuses to read keys from the settings file unless you opt in with
"allow_secrets_in_settings_file": true.
